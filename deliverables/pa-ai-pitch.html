Three capabilities are architecturally complete in this prototype. They are not yet connected to live systems — but the design is proven and activation is straightforward.

Designed + Built

① Eval Loop with Human-in-the-Loop

A Teaching Assistant pattern that quality-controls every AI decision. A Golden Set of expert-labeled cases acts as the answer key. A Grader agent compares each AI decision against the golden answer. When scores diverge on high-stakes cases, the system automatically escalates to a human nurse reviewer — with full context pre-loaded.

80% deterministic (zero tolerance) · 10% autonomous (scored on 3 dimensions) · 10% already escalated by clinical complexity

Eval Flow
AI Decision
↓
Grader compares to Golden Set
↓
MATCH → Auto-finalize
BORDERLINE → Flag for review
MISMATCH → Nurse escalation
Designed + Built

② Knowledge Wiki — Engineering Project Context

Clinical guidelines and business rules live as structured Markdown files — readable by non-engineers. When a payer changes a coverage policy, the clinical team edits one file. The AI pipeline picks it up immediately. No deployment. No code change.

This same pattern scales to the AI engineering team: system context, onboarding docs, domain knowledge — all versioned, searchable, and injected into AI agents at runtime.

③ Google ADK — Enterprise Deployment Path

The same Claude prototype is mirrored as a Google Agent Development Kit (ADK) project using Gemini 2.5 Pro on Vertex AI. This means: enterprise auth, SOC 2 compliant infrastructure, auto-scaling, and production observability — without rewriting any business logic.

The SKILL.md files are identical in both. Swapping from Claude to ADK/Gemini is an infrastructure decision, not a logic rewrite.

Why SKILL.md Files

Business rules are written in plain English by clinical staff, version-controlled by engineers, and injected into AI agents at runtime. No re-training. No model fine-tuning. Update a file → new rule in effect.

Domain Expansion = Add a Folder

Today: RA Biologic (complete) + 4 stubs. Adding Oncology means writing one SKILL.md. The router auto-discovers it. Parallel multi-agent processing across domains is architecturally in place.
